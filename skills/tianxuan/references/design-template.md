## 填写要点

- **按需选择**：纯后端 API 需求只做架构设计；纯 UI 调整只做 UI 原型；全栈新功能三项都做
- **探索现有代码先行**：先读代码结构再设计，不凭空设计
- **提 2-3 方案带 trade-offs**：不直接给单一方案
- **约束要可机检**：架构决策落地为可机检约束（不说"用JWT"，要说"认证返回JWT，过期2h，字段token"）
- **不拆 Task**：天璇只设计，拆解归天玑
- **变更记录**：洞明打回/瑶光路由/需求变化时追加版本
