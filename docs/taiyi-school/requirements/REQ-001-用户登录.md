# 需求真言

**混沌原念**: "我要给系统加个登录功能。"

**我假设的（已确认/已纠正）**:
1. 这是给终端用户用的（非管理员后台）——用户确认
2. 登录方式是手机号 + 密码（非短信验证码/第三方 OAuth）——用户确认，MVP 不做第三方
3. 密码需加密存储——用户确认

**所求之道**: 用户能用手机号和密码登录系统，获得身份凭证以访问受保护资源。

**筑基之痛**: 系统现有接口无身份校验，任何人可直接访问，存在越权风险。需要一个登录入口让用户获取凭证。

**真需求**: 手机号 + 密码的登录认证，返回可复用的身份凭证（token）。

**伪需求（已斩）**:
- 短信验证码登录 —— MVP 不需要，增加短信网关依赖，后续 Epic 可加
- 第三方 OAuth（微信/GitHub）—— 同上，后续 Epic 可加

**Not Doing**:
- 注册功能 —— 不在本次范围，假设用户已存在
- 密码找回 —— 同上，延后
- 多因素认证(MFA) —— 同上，标记为后续 Epic

**未明之境**: token 的过期策略（2h? 24h?）需天璇定，倾向 2h + refresh token。

**用户确认**: 已确认，此真言为所求之道。
