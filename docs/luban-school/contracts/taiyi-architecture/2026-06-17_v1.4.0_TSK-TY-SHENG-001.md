## 三、STORY 拆解

### STORY-1：司衡定位为"可选自动路由层"
- 在 SKILL 开头明确：司衡是可选增强层，九星 SKILL 内部只按手动模式设计。
- 删除正文中"司衡会自动唤醒你""等待司衡调度"等依赖描述。

### STORY-2：删除自动唤醒九星的强依赖
- 删除"自动续接机制"中"星最后一步调用 Skill("siheng")"的描述。
- 修改"五指规则"：手动模式下输出"请手动 @X星"，仅在环境支持时才尝试自动唤醒。
- 修改九星能力卡片标题与说明，去除"司衡流水线内调度""驱动唤醒"等强依赖暗示。

### STORY-3：增加手动召唤提示的降级路径
- 自动唤醒失败时一次性降级："当前应执行下一步：{manual_invoke}。当前环境不支持自动唤醒，请手动召唤。"
- 明确司衡只读 status.md/queue.md，不写任何文件。
